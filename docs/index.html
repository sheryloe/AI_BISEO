Service-facing project

AI 비서 운영과 블로그 자동화를 하나의 서비스 루프로 묶습니다

AI_BISEO는 채팅 응답, Notion 운영 로그, n8n 연동, 블로그 작업을 따로 흩어두지 않고 운영 콘솔 하나에서 확인하려는 프로젝트입니다. 핵심은 “AI가 뭘 했는지”를 남기고, “실패하면 어디서 막혔는지”를 바로 볼 수 있게 만드는 것입니다.

저장소 보기 이슈 정리

Core

AI 비서 + 블로그 자동화

Data

Prompt log + Notion 기록

Goal

운영형 서비스 콘솔

Assistant

채팅 응답과 운영 보조 기능을 같은 맥락에서 관리합니다.

Ops Log

Prompt log와 Notion 메모를 운영 증적처럼 쌓아 둡니다.

Pipeline

n8n, Telegram, 블로그 작업을 한 루프로 연결합니다.

Ops Ready

실패 추적, 환경 변수, Docker 흐름까지 같이 챙깁니다.

Workflow

운영 루프는 네 단계로 정리됩니다

AI_BISEO가 잘 되려면 “잘 답변한다”보다 “운영자가 상태를 이해한다”가 더 중요합니다. 그래서 생성, 기록, 자동화, 모니터링이 끊기지 않게 이어지는 구조를 강조합니다.

01 Generate

AI 비서 응답과 자동화 명령을 실행합니다.

02 Record

Prompt log와 Notion 기록으로 작업 흔적을 남깁니다.

03 Automate

n8n, Telegram, 블로그 파이프라인을 연결합니다.

04 Operate

실패 구간과 상태를 운영자가 바로 볼 수 있어야 합니다.

Security

권한과 자격증명 분리

운영용 API 키, 자동화용 토큰, 로컬 테스트 키를 분리하고 접근 범위를 줄여야 합니다.

Product

프롬프트 버전 관리

어떤 프롬프트 조합이 어떤 결과를 냈는지 비교할 수 있어야 서비스 개선 속도가 빨라집니다.

Ops

실패 재시도와 알림

자동화가 막혔을 때 즉시 알려주고, 재시도 기준을 명확히 두는 것이 다음 단계입니다.
