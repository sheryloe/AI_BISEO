# AI_BISEO 노션 페이지 구조 (3-Step)

상위 페이지 제목은 **AI_BISEO | AI 비서 운영 서버 제작기** 로 두고, 아래 3개 하위 페이지를 넣으면 됩니다.

## 권장 설명

개인 AI 비서, 블로그 자동화, Notion 운영 로그를 하나의 Ops Console로 묶어 가는 과정을 3단계로 정리한 노션 원고 세트입니다.

## 권장 페이지 트리

- Step 1. Docker 중심 운영 아키텍처를 먼저 고정한 이유
- Step 2. Express와 Ops Console로 운영 루프를 만든 과정
- Step 3. 모듈 공통 관리 레이어와 블로그 자동화 파이프라인 확장

## 스토리 라인

- Step 1에서는 기능보다 Docker 기준 운영 구조와 환경 분리를 먼저 고정한 이유를 설명합니다.
- Step 2에서는 Express, Socket.io, 대시보드를 연결해 실제 운영 루프를 만든 흐름을 보여 줍니다.
- Step 3에서는 모듈 공통 API와 AI Writer 파이프라인 추적 구조로 확장된 결과를 정리합니다.
